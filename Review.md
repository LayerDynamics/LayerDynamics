# Code Review: LayerDynamics

> Full-project, git-free review. Date: 2026-06-03.
> Method: direct reads + empirical `tsc`/`test`/`lint` runs (portal), plus 6 parallel read-only review agents across the scene layer, DOM/routes/contact form, state/build config, CI/profile-gen/infra, the Blender asset pipeline, and the workspace stubs. All findings cite `file:line`.

## Summary

LayerDynamics is **two products sharing one repo**: (1) a published **GitHub profile** (README + SVGs regenerated by GitHub Actions from the GitHub API), and (2) an in-progress **interactive 3D portfolio monorepo** — a scroll/level-driven React 19 + React-Three-Fiber WebGL site (`apps/client`, the one real runnable app, ~5,000 LOC), a two-sided **portal provider** package that windows live cross-origin apps into the 3D scene (`packages/portal`, at M0 foundations), a **Blender→glTF asset pipeline** (Python/bpy + a localhost FastAPI generation service), and a set of **intentionally-empty workspace stubs** (`packages/*`, `crates/*`). It deploys on Railway via Docker.

Overall the live code is well-architected and unusually well-commented: the level system, mobile/contain-fit framing, zustand selector discipline, and headless-WebGL test contract are genuine strengths. The dominant cross-cutting problems are **(a) write-capable secrets inlined into the public client bundle**, **(b) a large dead "continuous-scroll" architecture left mounted-nowhere alongside the live level system**, **(c) a contact form that reports false success when misconfigured**, and **(d) `strict` mode absent from the client TypeScript config**. The portal package's own `pnpm lint` gate is currently red.

## Findings

### Critical

- **Discord webhook URL inlined into the public client bundle** (`Dockerfile:29-32`, `apps/client/.env`, `apps/client/src/components/HireMe/submit.ts:11-12,56`) — `VITE_DISCORD_WEBHOOK_URL` is a build-time `ARG`/`ENV` that Vite inlines into the static JS shipped to every visitor. A Discord webhook URL is a **write-capable secret**: anyone who extracts it from the bundle can POST arbitrary messages to that channel (spam/phishing/abuse) until it is rotated. Unlike the Web3Forms access key (a genuine public submission key), the webhook does not belong client-side. Proxy Discord posts through a server endpoint (the portal provider could host it) instead of inlining the URL, and rotate the current webhook.

## Strengths

- **Mobile/portrait framing is a first-class, well-reasoned design** — `LevelCamera.tsx:33-55` does a true contain-fit (max of width/height-fit distance) instead of fov-fisheye; `OtherWorkLevel` `responsiveGrid` drops columns on narrow aspects while holding world width constant and divides card turn-in rotation by `cardScale` so big mobile cards don't over-rotate. Directly satisfies the repo's mobile non-negotiable.
- **Disciplined zustand + cross-Canvas state** — every `useScene` setter returns the same object when unchanged (no-op-render guard), selectors are atomic, and per-frame scroll uses a module singleton (`levelScroll.ts`) deliberately kept out of React to avoid 60fps re-renders.
- **`useLevels` is a clean, data-driven, genuinely-tested reducer** — pure transition state machine with re-entrancy guards and clamps; adding a level is a one-line data edit. The test exercises the real reducer.
- **Robust headless-WebGL test contract** — `sceneTest.ts` asserts cumulative `gl.info.render.calls > 0` (catches blank/off-screen frames a "canvas present" check would miss); Storybook `main.ts` `optimizeDeps` + `resolve.dedupe:['three']` solve the real optimizer-race / multiple-three-instances problems with documented rationale.
- **Solid contact-form a11y and no injection surface** — labels + `aria-invalid`/`aria-describedby`, `role="alert"`/`status`, no `dangerouslySetInnerHTML`, `encodeURIComponent` mailto, `rel="noreferrer"` external links; smart/presentational split with a pure, testable `validateHireMe`.
- **Memory-conscious scene teardown** — `LevelStage` keys Suspense per level id to force real unmount/dispose with one level mounted at a time; `useGLTF.preload` + `dispose={null}` correctly avoids disposing the shared drei-cached GLTF.
- **Portal contract layer is exemplary** — `shared/contract.ts` is the single DOM/Node-agnostic source of truth with string-union enums, canonical `ROUTES` builders, and a real runtime guard; allowlist is enforced in depth (route 403 + negotiate→null + serve only-known); `/config` strips `origin`/`sandbox` with a test asserting no leak; provider tests are real (`app.inject` + a live upstream Fastify proxy), not mocked.
- **CI/profile-gen security baseline is sound** — no workflow-injection surface (no untrusted `${{ }}` into `run:`), least-privilege `permissions: contents: write`, bounded `timeout-minutes`, all API-derived strings `escape_xml`'d into the SVG, no auto-commit recursion, clean multi-stage Docker (only `dist` shipped, pinned `pnpm@10.32.1`, `--frozen-lockfile`), `.npmrc` carries no secrets, and the Vite preview `allowedHosts` Railway gotcha is already handled.
- **Blender pipeline is honestly and precisely documented** — argv-list subprocesses (no `shell=True`), `os.path.basename` traversal defense on `/result` + `/upload`, localhost bind + single-job busy-lock, data-driven axis derivation in `30_rig.py`, and machine-checkable `@@TAG`/assert checkpoints. Hard-won macOS/OCP/Cycles gotchas are recorded in-comment, which materially aids reproducibility.
- **Workspace manifests are perfectly consistent** — all seven `packages/*` `package.json` are valid, `@layerdynamics/<dir>` name matches directory, `private`, `type:module`, `0.0.0`, no version drift.

## Recommendations (prioritized)

1. **Rotate and de-inline the Discord webhook** (Critical). Move both Discord and Web3Forms posts behind a tiny server endpoint; stop inlining write-capable URLs into the client bundle.
2. **Fix the contact-form false-success** (High): make the mailto fallback report "draft opened — please send," and split the success copy by which delivery path ran.
3. **Turn on `strict` in the client tsconfig** (High) via an extracted `tsconfig.base.json`, and add a `tsconfig.storybook.json` so `.storybook`/config files are type-checked.
4. **Harden the Blender service** (High): `TrustedHostMiddleware` + Origin allowlist (DNS-rebinding), confine `image_path` under `UPLOAD_DIR`, and bound `/upload` size.
5. **Delete (or quarantine) the dead continuous-scroll island** (Medium): `SceneContent`, `CameraRig`/`useCameraRig`, `ProjectCollection`, old `Contact`, `LensToggle`, and `useScene.section` — but verify `GlassLayer`/`LogoSpin`/`ProjectAccent`/`contactY` stay (they're live).
6. **Portal: consume `allowedOrigin`** (fixes the security no-op + the red lint gate in one change), fix the per-connection WS `close` cleanup, drop the `get()` global fallback, and add the `PORTAL_DATA`↔`REGISTERED_APPS` drift test.
7. **Fix GPU leaks** (Medium): dispose the `PrintedTitle` material on unmount; gate the invisible `LayeredBackdrop` glow behind `glowOpacity > 0`.
8. **CI/infra hardening** (Medium): add `.env` to `.dockerignore`, pin Actions to SHAs, add `urlopen(timeout=)` + per-repo try/except, `USER node` in the Dockerfile.
9. **Add the missing Printer/PrintedTitle stories**, fix the no-op Storybook theme decorator + stale violet backgrounds, and finish the Blender README/script reconciliation.
10. **Clean the package stubs**: empty `raycast`/`stylekit` `src/`+`index.html`, remove or populate the 0-byte tsconfigs/`index.html`, standardize the stylekit tsconfig name, and correct the stale CLAUDE.md root-`package.json` note.
